java-红黑树

Posted on 2019-09-22

1.特性
(1）节点是红色或黑色。

(2）根节点是黑色。

(3）每个叶子节点都是黑色的空节点（NIL节点）。

(4）每个红色节点的两个子节点都是黑色。(从每个叶子到根的所有路径上不能有两个连续的红色节点)

(5）从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点。

2.目的：解决二叉查找树多次插入不平衡，退化为链表

3.缺陷：当插入或者删除节点的时候，红黑树的规则可能被打破

4.解决方法

（1）变色

（2）旋转：左旋转+右旋转

5.插入

—–》》插入的节点为红色，如果是黑色，调整起来比较麻烦，有以下五种情况，这里假设要插入的节点为 N，N 的父节点为 P，祖父节点为 G，叔叔节点为 U。

（1）插入的节点N是红黑树的根节点，将N的颜色变为黑色。

（2）N的父节点是黑色，无需调整。

（3）N的父节点是红色，先将P和U染黑，再将G染红（若G此时和它的父节点形成连续的红色节点，需要递归向上调整）

（4）N 的父节点为红色，叔叔节点为黑色。节点 N 是 P 的右孩子，且节点 P 是 G 的左孩子。对P进行左旋，调整N和P的位置，染红按照情况5处理。

（5）N 的父节点为红色，叔叔节点为黑色。N 是 P 的左孩子，且节点 P 是 G 的左孩子。此时对G进行右旋，调整P和G的位置，并且互换颜色。

总结–插入：后三种的区别在于叔叔节点的颜色，若为红色，直接变色；若为黑色，先选择再交换颜色。

6.删除

原理：先找到该节点的前驱（该节点左子树中最大的节点）或者后继（该节点右子树中最小的节点），然后将前驱或者后继的值复制到要删除的节点中，最后再将前驱或后继删除。
